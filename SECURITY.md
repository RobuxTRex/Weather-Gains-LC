# Security Policy

## Supported Versions
WeatherGains is a Lethal Company mod intended for use in gameplay and educational environments. Security updates will generally **only be applied to the latest release**.
Older versions may and will **not receive patches**, so please upgrade to the most recent version when possible.

---

## Reporting a Vulnerability
If you discover a security vulnerability in WeatherGains (such as an exploit, data leak, or other unintended behaviour), **please do not create a public GitHub Issue under any circumstances**.

Instead, report it privately through one of the following channels:
- **Email**: [codesecurity@sulphurdev.co.uk](mailto:codesecurity@sulphurdev.co.uk)
- GitHub Security Advisories

When reporting, please include the following:
- A clear and detailed description of the vulnerability; don't leave out any details.
- Steps to reproduce the issue.
- The potential impact (e.g. exploitability, data exposure).
- Any possible mitigations you have identified.

---

## Responsible Disclosure
- Please **do not** publicly disclose vulnerabilities until they have been completely fixed and announced by the maintainer.
- Please **do not** attempt to exploit vulnerabilities in public environments.
- Coordinated disclosure is appreciated - we will acknowledge your report and keep you informed of progress.

---

## Security Response Process
1. Confirm receipt of your report (almost always within 72 hours).
2. Investigate the issue and assess severity.
3. Implement a fix and test thoroughly.
4. Release a patched version and update documentation accordingly.
5. Credit the reporter (if desired).

---

## Notes
- WeatherGains is intended for public and/or educational use; please do not attempt to use vulnerabilities for malicious purposes.
